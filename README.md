## How It Works

### User Journey:

**1. Submit Hypothesis**

```
User: "I think protein folding efficiency could be improved 
       if chloroplasts were arranged differently."

iQueue AI: "Let's make this testable. What specific arrangement 
            do you propose? What metric defines 'improved'?"
```

**2. Methodology Wizard**

```
Charter-based gates (automated):
âœ“ G1: Numeric thresholds defined?
âœ“ G2: Bounded scope (one claim)?
âœ“ G3: Operational definitions clear?
âœ“ G4: Success/failure criteria locked?
âœ“ G5: Negative controls specified?
âœ“ G6: Reproducibility parameters set?

AI guides user to satisfy each gate.
Can't proceed until all gates pass.
```

**3. Preregistration Lock**

```
Complete specification:
- Hypothesis (falsifiable claim)
- Methodology (how we'll test)
- Success criteria (what proves you right)
- Failure criteria (what proves you wrong)
- Negative controls (what validates the test)

Cryptographic hash generated.
Timestamp locked.
Methodology is now frozen.
```

**4. Queue Simulation**

```
Simulation queued on cloud infrastructure.
Progress updates in real-time.
Estimated completion time shown.
Can queue multiple simulations.
```

**5. Results & Validation**

```
Automated checks:
- Simulation ran correctly?
- Results meet preregistration criteria?
- Negative controls behaving as expected?
- Quality gates passed?

Output: PASS / FAIL / INCONCLUSIVE
Full provenance package generated.
```

**6. Publishing**

```
Validated result â†’ Citable repository created
DOI assigned
Full reproducibility package
Share link, download data

Failed results also published.
(Negative results advance knowledge too.)
```
